How It Works
Your entire skill ecosystem,
one conversation away.
CoCo is a persistent Node.js process that sits between you and Claude Code. You talk; it figures out which of your 75 skills to invoke.
The Flow
You speak or type → CoCo classifies → skill executes
  • Input — Text via terminal or voice via push-to-talk (whisper.cpp)
  • Classify — Tier 1 checks corrections, slash commands, regex, keywords (instant). Tier 2 falls back to Claude for ambiguous inputs.
  • Route — Team/GSD routers extract params. Orchestrator spawns a Claude Code session with the right skill context.
  • Execute — Session runs in a collapsible dashboard panel with live output. Multiple sessions run in parallel.
  • Learn — Every classification is logged. Corrections (/correct) override future routing. Skill sequences are tracked for recommendations.
Two Modes
Interactive TUI or single-command

Interactive: Full Ink terminal UI with dashboard, status bar, suggestion bar, session panels, voice toggle. Launches with coco or npm run dev.

Single-command: Pipe one command and exit. coco "research OAuth patterns" — output streams to stdout, no TUI needed.

Proactive Mode

When enabled (/proactive on), CoCo watches your files, email (HxStore), and calendar (AppleScript) — suggesting actions in a non-intrusive bar. All suggestions require explicit Enter to accept. Toggle per channel: /proactive email on, /proactive calendar on.

Core Features
Built for how you actually work.
Every feature designed around real daily workflows — multi-tasking, voice, intelligence, proactive suggestions, and the full skill ecosystem.
Two-Tier Intent Classification

Tier 1: corrections store, slash commands, regex triggers, keyword scoring (instant). Tier 2: Claude-based for ambiguous inputs. 100% on 50-query test set.

Parallel Sessions + Dashboard

Run 3+ Claude sessions simultaneously. Each gets a collapsible panel with live output, status, layer progress, and elapsed time.

Voice I/O (Offline)

Push-to-talk via local whisper.cpp. TTS via macOS say. No cloud APIs. Toggle with /voice on.

Learning Loop

Say /correct team-review and CoCo learns. Corrections stored in SQLite, applied first on future inputs. Skill sequence recommendations after 3+ occurrences.

Proactive Suggestions

File watcher, email monitor (HxStore + attachments), calendar bridge. Non-intrusive suggestion bar — accept with Enter, dismiss with Esc. Never auto-executes.

75 Skills Auto-Discovered

Reads ~/.claude/commands/ at startup. Covers /team, /gsd, /pmstudio, /email, Superpowers, and custom commands.

Task Queue + Dependencies

Queue commands while sessions run. FIFO with dependency support and priority. "After the research, build the component."

Ambient Context

Greets with project name, branch, last session, queue depth. Summarizes exchanges every 10 turns. Manages token budget automatically.

Persistent SQLite State

15+ tables: sessions, history, skills, queue, intent log, corrections, sequences, summaries, preferences, suggestions. Survives restarts at ~/.coco/coco.db.

Quick Start
Up and running in minutes.
Prerequisites
  • Node.js 20+ installed
  • Claude CLI authenticated (claude -p works)
  • macOS (voice uses say / whisper.cpp)
Voice Setup (Optional)

Install whisper.cpp via Homebrew for push-to-talk voice input. CoCo downloads the base English model automatically on first /voice on.

Fully Offline
# Navigate to CoCo
cd ~/Downloads/coco/src

# Install dependencies
npm install

# Interactive TUI (full dashboard)
npm run dev

# Single-command mode (no TUI needed)
npx tsx src/index.tsx "research OAuth patterns"

# Global install (use `coco` anywhere)
npm run link

# Run all 227 tests
npm test

Why CoCo?
Nothing else does this.
14 tools evaluated. None combine: always-on daemon + voice + terminal orchestration + extensible skills + learning loop.
Key Insight
34
source modules

34 TypeScript modules across 6 directories. 227 tests. 15+ SQLite tables. From intent classification to proactive file watching — all built, verified, and code-reviewed.

  • Claude Code — Skills + hooks + MCP + 1M context. No daemon, no voice.
  • OpenAI Codex CLI — Voice input (experimental). OpenAI only, no daemon.
  • Aider — Best git integration, voice. Code-only, no general tasks.
  • Open Interpreter — General-purpose execution. AGPL, no daemon, no MCP.
  • TmuxAI — Observe all panes passively. Observation only, no execution.
